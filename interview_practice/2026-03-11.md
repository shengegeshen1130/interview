### Q2 (12:00) - KV Cache原理+PagedAttention
**题目：** KV Cache是什么、为什么需要、内存占用计算、PagedAttention如何优化KV Cache管理。
**shen的回答：** 未作答（直接要答案）

**参考答案：**
1. **KV Cache是什么：** 自回归生成时，每个新token需要和所有历史token做attention。如果不缓存，每步都要重新计算所有历史token的K/V向量（重复计算量巨大）。KV Cache把已计算的K/V存下来，每步只算新token的Q/K/V，然后把新K/V追加到cache。
2. **为什么需要：** 没有KV Cache时生成N个token的复杂度是O(N³)（每步重算1+2+...+N）；有KV Cache后降到O(N²)。是推理加速的基础优化。
3. **内存占用计算：** 每层每token存K和V各一个向量。公式：KV Cache大小 = 2 × n_layers × n_kv_heads × d_head × seq_len × dtype_bytes。例：Llama-2-7B（32层, 32 heads, d=128, FP16）seq_len=4096 → 2×32×32×128×4096×2B = 2GB/请求。这是推理显存的主要开销。
4. **PagedAttention优化：** 传统KV Cache按最大序列长度预分配连续内存→碎片严重。PagedAttention将KV Cache切成固定大小block（如16 tokens），按需分配非连续物理块，通过block table做逻辑→物理映射。好处：①显存碎片几乎为零；②支持copy-on-write（beam search共享block）；③动态释放已结束序列的block。

### Q4 (14:00) - Transformer细节（Pre-LN/RMSNorm/RoPE/SwiGLU）
**题目：** Pre-LN vs Post-LN区别、RMSNorm vs LayerNorm、RoPE位置编码原理、SwiGLU激活函数。
**shen的回答：** 已作答（4/10，迟答），详细回答未记录。

**参考答案：**
1. **Pre-LN vs Post-LN：** Post-LN（原始Transformer）= Sublayer → Add → LN，梯度在深层爆炸/消失，需要warmup。Pre-LN = LN → Sublayer → Add，梯度传播更稳定，几乎所有现代LLM（GPT/Llama/Mistral）都用Pre-LN。代价：Pre-LN最终层输出scale可能偏小，有些工作加一层额外LN。
2. **RMSNorm vs LayerNorm：** LayerNorm: y = (x - μ) / √(σ² + ε) × γ + β，需要算均值和方差。RMSNorm: y = x / √(mean(x²) + ε) × γ，去掉了减均值和加偏置β，只用RMS归一化。优势：计算量减少~10-15%，效果几乎无损。Llama全系列使用RMSNorm。
3. **RoPE位置编码：** 将位置信息编码为旋转矩阵，对Q/K向量做旋转变换。核心性质：位置m的Q和位置n的K做内积时，结果只依赖相对位置(m-n)，天然支持相对位置编码。公式：将d维向量两两一组，每组旋转θ_i·m角度（θ_i = 10000^(-2i/d)）。优势：①外推性好（NTK-aware scaling可扩展到训练长度之外）；②无需学习参数；③天然支持线性attention。
4. **SwiGLU激活函数：** FFN(x) = (Swish(xW₁) ⊙ xV) · W₂。Swish(x) = x·σ(βx)。相比ReLU：门控机制(⊙)让网络选择性传递信息，实验表明SwiGLU比ReLU/GELU效果好但参数量多~33%（多一个V矩阵）。Llama/PaLM均使用SwiGLU。

### Q5 (15:00) - 量化（INT8/GPTQ/AWQ）
**题目：** 量化基本原理、INT8量化方式、GPTQ和AWQ的核心思想、量化对模型精度的影响。
**shen的回答：** 未作答（直接要答案）

**参考答案：**
1. **基本原理：** 将FP16/FP32权重映射到低精度（INT8/INT4），减少显存和计算量。线性量化：q = round(x / scale + zero_point)，反量化：x̂ = (q - zero_point) × scale。per-tensor vs per-channel vs per-group（group_size=128最常见）。
2. **INT8量化：** ①W8A8（权重和激活都INT8）：LLM.int8()（Dettmers），发现激活中有少量outlier channels值域极大，混合精度处理（outlier用FP16，其余INT8）；②W8A16（权重INT8，激活FP16）：更简单，精度损失小。
3. **GPTQ：** 后训练量化（PTQ），逐层量化。核心：利用Hessian矩阵（二阶信息）确定量化顺序——先量化对输出影响最小的权重列，量化误差通过剩余未量化列补偿（OBQ思想）。优势：INT4量化后精度损失极小；只需少量校准数据（128条）；单次forward完成，速度快。
4. **AWQ（Activation-aware Weight Quantization）：** 观察到权重中有少量"显著通道"（对应激活值大的channel），这些通道量化误差影响巨大。方法：对显著通道乘以scale放大后再量化（等效于缩小其量化误差），scale通过激活分布自动确定。比GPTQ更简单（不需要Hessian），速度更快，效果相当。
5. **精度影响：** INT8几乎无损（<0.5% perplexity退化）；INT4通常1-3%退化；INT3开始明显退化。小模型量化损失比大模型更严重。GPTQ/AWQ的INT4通常好于naive INT8。

### Q6 (16:00) - 对比学习+Embedding模型训练
**题目：** 对比学习原理、InfoNCE损失、Embedding模型训练流程（负采样/hard negative）、评估指标。
**shen的回答：** 已作答（4/10，迟答），详细回答未记录。

**参考答案：**
1. **对比学习原理：** 让相似样本（正对）在嵌入空间中靠近，不相似样本（负对）远离。核心思想：不直接学标签，而是学样本间的相对关系。自监督场景下，同一样本的不同增强版本构成正对。
2. **InfoNCE损失：** L = -log(exp(sim(q,k⁺)/τ) / Σ exp(sim(q,kᵢ)/τ))。τ是温度参数（越小分布越尖锐，对hard negative更敏感）。本质是(N+1)分类的交叉熵——把正样本从N个负样本中区分出来。负样本越多、越难，学到的表示越好。
3. **Embedding模型训练流程：**
   - *数据构建：* (query, positive_doc, hard_negative_doc) 三元组。Hard negative = BM25/向量检索召回的高分但不相关文档（比随机负样本有效10x+）。
   - *模型架构：* Bi-Encoder（双塔），query和doc独立编码，用cosine similarity。通常在BERT/E5/BGE基础上微调。
   - *训练技巧：* in-batch negatives（同batch其他正样本的doc互为负样本，免费扩大负样本量）；curriculum learning（先easy negatives再hard negatives）；knowledge distillation（用Cross-Encoder teacher打分指导Bi-Encoder student）。
4. **评估指标：** Recall@K、MRR（Mean Reciprocal Rank）、NDCG@K、Hit Rate。MTEB benchmark是Embedding模型标准榜单。

### Q8 (18:00) - CoT/ToT/ReAct推理框架
**题目：** Chain-of-Thought原理、Tree-of-Thoughts扩展、ReAct框架、三者适用场景对比。
**shen的回答：** 未作答

**参考答案：**
1. **Chain-of-Thought (CoT)：** 通过"Let's think step by step"或few-shot示例引导模型输出中间推理步骤再给最终答案。原理：将复杂问题分解为多步简单推理，每步利用前一步结果。Zero-shot CoT只需加"Let's think step by step"；Few-shot CoT提供带推理过程的示例。局限：线性推理，一条路走到底，如果中间某步错了就会连锁错误。
2. **Tree-of-Thoughts (ToT)：** 将CoT的线性链扩展为树状搜索。每步生成多个候选thought，用LLM评估每个thought的价值（自评分），通过BFS/DFS搜索最优路径。支持回溯——如果某条路走不通，退回上一步换另一条。适合：需要探索和回溯的推理任务（24点游戏、创意写作规划）。代价：推理开销大（多次LLM调用）。
3. **ReAct（Reasoning + Acting）：** 交替进行Thought（推理）→Action（调用工具）→Observation（获取结果）循环。Thought：模型分析当前状态、规划下一步；Action：调用搜索/计算器/API等工具；Observation：工具返回结果，模型继续推理。解决了CoT不能获取外部信息的问题。是Agent的基础框架。
4. **对比与适用场景：**
   - CoT：纯推理任务（数学/逻辑），不需要外部信息，一次调用
   - ToT：需要探索多条路径的复杂推理，允许回溯，代价是多次调用
   - ReAct：需要与外部世界交互（搜索/API/数据库），Agent任务标配
   - 实践中常组合使用：ReAct框架内每步用CoT推理，复杂决策用ToT搜索

### Q9 (19:00) - 知识蒸馏（KD）+DeepSeek-R1蒸馏
**题目：** 知识蒸馏原理、soft label vs hard label、DeepSeek-R1的蒸馏方案、蒸馏在LLM中的应用。
**shen的回答：** 未作答

**参考答案：**
1. **知识蒸馏原理：** Teacher（大模型）的软概率分布包含类间关系信息（如"猫"和"虎"的概率都高→形态相近），Student（小模型）学习这个分布比学hard label（one-hot）能获得更多信息。损失函数：L = α·L_hard(y, p_s) + (1-α)·T²·KL(p_t/T || p_s/T)。T是温度，升高T使分布更平滑，暴露更多类间关系。
2. **Soft label vs Hard label：** Hard label=one-hot标签，只告诉"正确答案是A"；Soft label=Teacher的概率分布[0.7, 0.2, 0.05, 0.05]，额外告诉"B也有些像A"。Soft label信息量更大，且有正则化效果（防止Student过度自信）。Hinton 2015原始论文。
3. **DeepSeek-R1蒸馏方案：**
   - *两阶段：* ①先用RL（GRPO）训练大模型R1获得强推理能力（长CoT）；②用R1的推理输出（包含完整思维链）作为训练数据，SFT蒸馏到小模型（1.5B/7B/8B/14B/32B/70B）。
   - *关键发现：* 蒸馏效果 > 小模型直接做RL。R1-Distill-Qwen-32B在数学/代码benchmark上超过OpenAI o1-mini。
   - *蒸馏数据：* 约80万条R1生成的(问题, 长推理过程+答案)对，包含数学/代码/推理任务。
4. **LLM蒸馏应用：** ①Alpaca/Vicuna：GPT-4蒸馏到LLaMA；②Orca：学习GPT-4的推理过程（不只是答案）；③DistilBERT：保留97%性能但快60%。蒸馏是当前最实用的小模型能力提升手段。

### Q10 (20:00) - 采样策略（Beam/Top-K/Top-P/Temperature）
**题目：** 各采样策略原理、Temperature作用、Beam Search vs 采样、Agent vs 创意写作推荐配置。
**shen的回答：** 未作答

**参考答案：**
1. **Greedy Decoding：** 每步选概率最高的token。确定性但单调、容易重复。
2. **Temperature：** p_i = exp(logit_i / T) / Σ exp(logit_j / T)。T<1分布变尖锐（更确定）；T>1分布变平坦（更随机）；T→0退化为greedy；T→∞退化为均匀随机。本质是调节softmax的entropy。
3. **Top-K：** 只保留概率最高的K个token，重新归一化后采样。问题：K固定，有些位置分布集中（只需K=5），有些位置分布分散（需要K=100），一刀切不灵活。
4. **Top-P（Nucleus Sampling）：** 动态选择最小token集合使累积概率≥P。自适应——分布集中时候选少，分布分散时候选多。比Top-K更灵活，是当前主流方法。
5. **Beam Search：** 维护B条候选序列（beam），每步扩展所有可能、保留总概率最高的B条。得到全局近似最优序列。适合翻译/摘要（需要高质量确定输出），不适合开放生成（输出boring/重复）。
6. **推荐配置：**
   - *Agent/工具调用：* T=0或T≈0.1，top_p=1，需要确定性和准确性
   - *创意写作：* T=0.7-1.0，top_p=0.9-0.95，需要多样性
   - *代码生成：* T=0.2-0.4，top_p=0.95，平衡正确性和多样性
   - *常见组合：* Temperature + Top-P同时使用（先调T再截Top-P）

### Q11 (21:00) - 数据工程（去重/过滤/Self-Instruct/DoReMi）
**题目：** 预训练数据清洗流程、去重方法、Self-Instruct数据生成、DoReMi数据配比。
**shen的回答：** 未作答

**参考答案：**
1. **预训练数据清洗流程：** 原始爬取（CommonCrawl）→ 语言识别（fastText）→ 规则过滤（长度/特殊字符/重复行比例）→ 质量过滤（perplexity filter用小LM打分/分类器判断高质vs低质）→ 去重 → 隐私过滤（PII removal）→ 毒性过滤。
2. **去重方法：**
   - *精确去重：* URL去重、文档hash去重（SHA256）
   - *近似去重：* MinHash + LSH（局部敏感哈希），将文档转为n-gram集合→MinHash签名→LSH分桶，同桶文档判为近似重复。Jaccard相似度阈值通常0.8。
   - *子串去重：* Suffix Array找重复n-gram段（CCNet方法），删除跨文档重复的长段落。
   - *重要性：* RedPajama发现去重后模型性能提升显著；重复数据导致memorization（隐私泄露风险）。
3. **Self-Instruct：** 用LLM自动生成instruction tuning数据。流程：①种子任务集（175条人写的）→②LLM生成新instruction→③LLM生成input→④LLM生成output→⑤过滤（去重/质量）→⑥加入种子集循环。Alpaca用Self-Instruct从GPT-3.5生成52K指令数据微调LLaMA。
4. **DoReMi：** 自动优化预训练数据的领域配比（代码/Wiki/Web/书籍各占多少）。方法：①训练一个小proxy模型；②用proxy模型的per-domain loss变化率确定最优权重（loss高的领域多给数据）；③用优化后的配比训练大模型。比人工调配比更有效，Llama等都用类似方法。

**题目：** 设计一个生产级AI客服Agent系统，包括架构、RAG、多轮对话、escalation、监控。

**参考答案：**
1. **整体架构：**
   - *接入层：* 多渠道接入（Web Chat/微信/电话）→ 统一消息格式 → 消息队列（Kafka）
   - *路由层：* 意图分类（FAQ/投诉/技术问题/需人工）→ 分发到对应处理pipeline
   - *Agent层：* LLM + RAG + Tools，处理用户请求
   - *人工兜底层：* Agent无法解决时转人工，附带对话摘要和上下文
2. **RAG设计：**
   - 知识库：产品文档/FAQ/历史工单，定期更新
   - Chunking：按FAQ条目+语义分段，保留metadata（产品线/版本/更新日期）
   - Hybrid Search（BM25+向量）→ Reranker → Top-3喂入LLM
   - 引用溯源：回答附带来源链接，便于验证
3. **多轮对话管理：**
   - 短期记忆：当前session全部消息
   - 上下文压缩：超过N轮时摘要历史（保留关键信息）
   - 槽位填充：订单查询需要订单号→引导用户提供
4. **Escalation策略：** ①Agent自评confidence < 阈值 → 转人工；②用户情绪检测（负面/愤怒）→ 优先转人工；③连续N轮未解决 → 自动转人工；④敏感话题（退款/投诉/法律）→ 规则强制转人工。转接时传递完整上下文摘要。
5. **监控与评估：** ①解决率（无需转人工的比例）；②用户满意度（对话后评分）；③响应延迟（P50/P99）；④幻觉率（抽样人工检查）；⑤每日成本（token用量）。告警：解决率下降/延迟飙升/成本异常。
6. **安全：** 输入过滤（prompt注入检测）；输出过滤（不泄露内部信息）；PII脱敏；audit log全量记录。
